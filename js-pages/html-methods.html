Мой Справочник | JavaScript | Методы работы с DOM

Селекторы

.querySelector()

 document.querySelector('.some-class'); Выбрать элементы по любому селектору как в CSS

Возвращает html-разметку со всеми вложенными элементами

.querySelectorAll()

 document.querySelectorAll('.some-class'); Выбрать все элементы по любому селектору как в CSS

.getElementById()

 document.getElementById('some-id'); Выбрать элемент по id

.getElementByName()

 document.getElementByName('some-id'); Выбрать элемент по атрибуту name

.getElementsByClassName()

 document.getElementsByClassName('some-class'); Выбрать элемент по классу

.getElementsByTagName()

 document.getElementsByTagName('p'); // all elements document.getElementsByTagName('*'); // first paragraph document.getElementsByTagName('p')[0]; Выбрать элементы по названию тега
DOM-"семейство"

parentNode

 .someElement.parentNode;
 Тоже что и parentElement, но не является массивом, это DOM-коллекция. Возвращает тоже самое, что и parentElement

.parentElement

 someElement.parentElement;
 Не функция.

Возвращает родительский элемент

.children

 someElement.children;
// первый дочерний элемент someElement.children[0];
 Возвращает массив всех дочерних элементов. К ним можно обращаться по индексу и применять все свойства для массивов

.childNodes

 someElement.childNodes.length;
// первый дочерний элемент someElement.childNodes[0];
 Что-то вроде children, но ведёт себя странновато, любые отступы в форматировании считает за дочерние элементы тоже, называя их #text. Не является массивом, это DOM-коллекция

.previousSibling

 someElement.previousSibling;
 Предыдущий "сосед"

.nextSibling

 someElement.nextSibling;
 Следущий "сосед"

.firstElementChild

 someElement.firstElementChild;
 Первый дочерний элемент

.lastElementChild

 someElement.lastElementChild;
 Последний дочерний элемент

.matches

 // содержит ли этот элемент селектор 'p' someElement.matches('p'); Проверка на совпадения селектора у какого-либо элемента. Возвращает bool

.closest

 // ближайший к элементу тег с классом '.some-class' someElement.closest('.some-class'); Ищет ближайший элемент вверх по цепочке

.contains

 someElement.contains(someElement2); Проверяет на наличе элемента внутри родителя. Возвращает bool
classList

.classList.add()

 someElement.classList.add('some-class'); Добавить класс к элементу someElement

.classList.remove()

 someElement.classList.remove('some-class'); Удалить класс у элемента someElement

.classList.toggle()

 someElement.classList.toggle('some-class'); Переключает класс у someElement на some-class

.classList.contains()

 someElement.classList.contains('some-class'); Проверка на наличие класса
Attributes

.setAttribute()

 someElement.setAttribute('style', 'color: red;'); Добавить атрибут или атрибут со значением

.hasAttribute()

 someElement.hasAttribute('style'); Проверка на наличие атрибута

.getAttribute()

 someElement.getAttribute('style'); Получить значение атрибута

.removeAttribute()

 someElement.removeAttribute('style'); Удалить атрибут

.className()

 someElement.className('some-class'); Добавить класс

.title()

 someElement.title('title text'); Атрибут title

.style

 someElement.style.color = 'red';
someElement.style.backgroundColor = 'red';
someElement.style.fontSize = '20px';
Атрибут style. Не функция. У каждого css-свойства есть свой метод в объекте style

.hidden

 someElement.hidden = true;
 Атрибут hidden. Не функция, скрывает, но не удаляет из DOM

.dataset

 <div data-filter="active"></div>
// поменять значение этого data-атрибута someElement.dataset.filter = 'no-active'; 		Дата-атрибут. Можно получить или изменить значение любых data-атрибутов
Создание новых элементов / Перемещение

.createElement()

 // создаём новый div var newElement = document.createElement('div');
// присваиваем класс newElement.className = 'class-1 class-2';
// и внутреннюю разметку newElement.innerHTML = '<b>New text</b>';
 Создание нового элемента

.cloneNode()

 // клонируем элемент var newElement = someElement.cloneNode(true);
// внутри него меняем текст тега p newElement.querySelector('p').innerHTML = 'New text';
 Клонирование элемента

.appendChild()

 someElement.appendChild(newElement);
 Вставить новый элемент в конец родителя

.removeChild()

 parentElement.removeChild(childElement);
 Удаление потомков

.replaceChild()

 parentElement.replaceChild(newChild, oldChild);
 Замена одного потомка на другой

.insertBefore()

 // второй арг. функции - перед каким элементом вставлять someElement.insertBefore(newElement, someElement.children[0]);
// если указать null, то сработает как appendChild someElement.insertBefore(newElement, null);
 Вставить новый элемент в начало родителя
forms

.forms

 docuement.forms;
// форма с id var someForm = docuement.forms['some-id'] // поля этой формы c id someForm['some-input-1'];
someForm['some-input-2'];
DOM-коллекция всех форм

.validity

 someElement.validity.ValidityState;
 Объект и подобъект, кот. хранят в себе валидацию

.value

 someElement.value = 2; Присваивание значений полям

.id

 someElement.id = 'some-id'; Значение id

.type

 someElement.type = 'email'; Тип поля

.min
.max

 someElement.min = 0;
someElement.max = 10; 		Макс. и минимальные значения полей

.selected

 someElement.selected = true;

.checked

 someElement.checked = true;
Работа с текстом и разметкой

innerHTML

 // change html-code someElement.innerHTML = '<p>Something'<p>;
 Разметка дочерних элементов. Не функция.

Работает с html-кодом. Воспринимает разметку

outerHTML

 // change html-code someElement.outerHTML = '<p>Something'<p>;
 Включает в себя всю разметку, по сравнению с innerHTML. Также возпринимает html-код

textContent

 // change only text someElement.textContent = 'Some text';
 Не функция.

Работает только с текстом. Разметка, вложенная в этот метод, будут парситься как текст
localStorage

.setItem()

 localStorage.setItem('name', 'John Smith'); Сохранить значение в хранилище.

name - переменная (ключ), внутри которой будет храниться 'John Smith'

.getItem()

 localStorage.setItem('name'); Получить значение ключа в хранилище.

Если у переменной name есть сохранённое значение в localStorage, то этот метод вернёт это значение

.removeItem()

 localStorage.removeItem('name'); Удалить значение ключа из хранилища.

Если у переменной name есть сохранённое значение в localStorage, то этот метод УДАЛИТ это значение

.clear()

 localStorage.clear(); Очистить всё хранилище.

Метод очищает все сохранённые значения в хранилище
Cookies

Сохранять значения в сookies

 document.cookie = 'myFirstCookie=Something'; // show all cookies alert(document.cookie); // создаём шаблон даты для cookie (3 дня) var dateToExpire = +Date.now() + 3*24*60*60*1000; // переводим её в читаемый вид для cookie var convertFormatDate = new Date(dateToExpire).toUTCString(); // сохранение значений полей document.cookie = 'inputVar=' + inputVar.value + ';expires=' + convertFormatDate;

Теперь нужно вставить эти сохранённые значения в поля при следуещем визите пользователя (В этом поможет библиотека cookies.js (Подключантся в <head>)).
Хранит cookie в объекте docCookies

// если в куках есть сохранённое значение, то оно вставиться в поле из кук, если в куках нет сохраннёных значений, то поле будет пустым inputVar.value = docCookies.getItem('inputVar');

В принципе, можно пользоваться только этой библиотекой и для сохранения в куки и для получения значений из кук (но лучше сохранять как в примере выше, так как сохранении кук этой библиотекой имеет проблемы с различными браузерами)

// сохранить куку (число-кол-во секунд) docCookies.setItem("cookie-name", "cookie-value", 150); // получить куку docCookies.getItem("cookie-name"); // удалить куку docCookies.removeItem("cookie-name"); // проверить куку docCookies.hasItem("cookie-name"); 		Сохраняем в ключ myFirstCookie значение Somethind. Эти сохранённые значения храняться на сервере тоже. Не более 4KB. Для больших объёмов нужно использовать localStorage

Чтобы сохранить cookie нужно при событии отправки (onsubmit),отправку остановить e.preventDefault(), затем сохранить введённые значения в куки, а потом отправить форму .submit()
