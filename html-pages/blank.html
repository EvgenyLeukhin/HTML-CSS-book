Мой Справочник | HTML | Семантика

  • <header>...</header>

    "Шапка" сайта. Контейнер. Может использоваться несколько раз. Может быть "шапкой" для section.

  • <footer>...</footer>

    "Подвал" сайта. Контейнер. Может использоваться несколько раз. Может быть "подвалом" для section.

  • <main>...</main>

    Основной контент. Контейнер.

  • <aside>...</aside>

    Сайдбар. Контейнер. Второстепенный контентный контейнер. Вспомогательный контент. Меньший приоритет у контента.

  • <nav>...</nav>

    Контейнер для навигационных элементов. Группы ссылок, ключевых навигационных элементов на сайте.

  • <section>...</section>

    Секция (часть) основного контента. Должен использоваться вместе с заголовком. Если заголовка нет, нужно его прятать (позиционированием или visibility). Не должен быть только обёрткой.

  • <article>...</article>

    Отдельный контент, не связанный с основным, может быть "оторван" от страницы и не потерять смысл. Должен использоваться вместе с заголовком. Котейнер.

  • <figure>...</figure>

    Контейнер для вставки какого-либо медиа-контента (изображение, флеш, видео и т.д.).

  • <strong>...</strong>

    Строчный элемент для смыслового выделения текста. Жирный текст.

  • <em>...</em>

    Строчный элемент для смыслового выделения текста. Курсив.

  • <details>...</details>

    Интерактивный разворачивающейся список

  • <summary>...</summary>

    Первый пункит этого списка, который всегда виден, второстепенный контент

  • <mark>...</mark>

    Выделение

  • <time>...</time>

    Дата и время

  • <h1>...</h1>

    Самый сильный сео-контейнер для текста заголовка. Лучше использовать 1 раз на странце.


Микроданные. Более глубокая семантика

Нужно для лучшей обработки результата поисковых запросов

itemscope - указывает границы блока по микроданным в открывающем теге

itemtype - тип схемы для микроданных с сайта http://schema.org/ (всегда URL-тип)

itemprop="name" - свойство этой схемы со значением (полный список значений на сайте http://ruschema.org/)

<div itemscope itemtype="http://schema.org/Person">
  <div itemprop="name">Евгений Леухин</div>
  <img itemprop="image" src="http://server/photo.jpg">
  <div itemprop="gender">Мужской</div>
  <div itemprop="url">http://mysite.ru</div>
  <div itemprop="birthDate">01-04-1986</div>
  <div itemprop="nationality">Русский</div>
  <div itemprop="homeLocation">Россия, Омск</div>
  <div itemprop="telephone">+79043270421</div>
  <div itemprop="email">evgenyleukhin@ya.ru</div>
  <div itemprop="jobTitle">Web-разработчик</div>
  <div itemprop="spouse">Бегунова Инна</div>
  ...
</div>
